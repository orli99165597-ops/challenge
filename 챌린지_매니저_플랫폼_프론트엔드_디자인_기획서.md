# 챌린지_매니저_플랫폼_프론트엔드_디자인_기획서

## 1. 문서 목적

이 문서는 `챌린지_매니저_플랫폼_기획서.md`와 `챌린지_매니저_플랫폼_최종_MVP_PRD.md`를 참고하여 작성한 프론트엔드/디자인 기획서입니다.

목표는 개발 전에 사용자 화면과 관리자 화면의 구조, UX 흐름, 주요 컴포넌트, 화면별 정보 우선순위를 명확히 정리하는 것입니다.

## 2. 디자인 방향

## 2.1 제품 성격

챌린지 매니저 플랫폼은 마케팅용 랜딩페이지보다 **실제 운영형 대시보드 서비스**에 가깝습니다.

따라서 첫 화면부터 사용자가 해야 할 행동이 명확해야 합니다.

- 사용자는 챌린지를 찾고, 참가하고, 오늘의 미션을 제출해야 합니다.
- 관리자는 챌린지를 만들고, 제출물을 검수하고, 보상 상태를 관리해야 합니다.

## 2.2 디자인 키워드

- 명확함
- 진행률
- 마감감
- 완주 동기
- 검수 효율
- 보상 신뢰
- 수익화 운영

## 2.3 디자인 톤

### 사용자 화면

사용자 화면은 “챌린지를 완주하고 싶다”는 동기를 주는 방향으로 설계합니다.

- 따뜻하지만 과하게 귀엽지 않은 톤
- 진행률과 마감 시간이 잘 보이는 UI
- 미션 제출 버튼이 명확한 구조
- 보상과 랭킹이 자연스럽게 보이는 구조

### 관리자 화면

관리자 화면은 운영자가 반복적으로 쓰는 업무 도구이므로 차분하고 밀도 있게 설계합니다.

- 표와 필터 중심
- 검수 대기, 지급 보류, 계좌 검수 등 상태가 한눈에 보여야 함
- 화려한 카드보다 리스트와 테이블 중심
- 클릭 수를 줄이는 구조

## 5. 사용자 화면 설계

## 5.1 로그인 화면

### 목적

사용자가 빠르게 로그인하거나 회원가입으로 이동합니다.

### 주요 요소

- 이메일 입력
- 비밀번호 입력
- 로그인 버튼
- 회원가입 이동
- 오류 메시지

### UX 기준

- 로그인 실패 시 오류 사유를 간단히 표시합니다.
- 관리자 계정도 같은 로그인 화면을 사용하되, 로그인 후 역할에 따라 관리자 화면으로 이동합니다.

## 5.2 회원가입 화면

### 주요 요소

- 이메일
- 비밀번호
- 비밀번호 확인
- 닉네임
- 약관 동의
- 가입 버튼

### 검증

- 이메일 중복 확인
- 비밀번호 최소 조건 안내
- 닉네임 필수 입력

## 5.3 메인 화면

### 목적

사용자가 현재 참여 가능한 챌린지와 내가 진행 중인 챌린지를 바로 확인합니다.

### 화면 구성

1. 상단 내비게이션
2. 오늘 진행 중인 내 미션 요약
3. 추천 챌린지 영역
4. 챌린지 유형별 빠른 진입
5. 랭킹 상위 콘텐츠 요약

### 주요 컴포넌트

- 오늘의 미션 카드
- 챌린지 카드
- 진행률 바
- 마감 시간 배지
- 보상 배지

### 챌린지 카드 정보

- 챌린지명
- 챌린지 유형
- 기간
- 참가비
- 보상 요약
- 참가자 수
- 상태: 모집중, 진행중, 종료
- 참가하기 버튼

## 5.4 챌린지 목록 화면

### 목적

사용자가 원하는 챌린지를 쉽게 찾습니다.

### 필터

- 전체
- AI 동화
- 블로그
- 유튜브
- 무료
- 유료
- 모집중
- 진행중
- 마감 임박

### 정렬

- 최신순
- 마감 임박순
- 참가자 많은순
- 보상 높은순

### UI

챌린지는 카드 그리드로 표시합니다. 모바일에서는 1열, 태블릿에서는 2열, 데스크톱에서는 3열을 기본으로 합니다.

## 5.5 챌린지 상세 화면

### 목적

사용자가 이 챌린지에 참가할지 결정할 수 있도록 핵심 정보를 제공합니다.

### 상단 정보

- 챌린지 제목
- 유형 배지
- 진행 기간
- 참가비
- 예상 보상
- 참가자 수
- 참가하기 버튼

### 본문 섹션

1. 챌린지 소개
2. 미션 일정
3. 제출 기준
4. 평가/랭킹 기준
5. 보상 기준
6. 유의사항

### AI 동화 챌린지 상세 예시

- 1일차-20일차: 매일 2쪽 제출
- 21일차: 전체 페이지 업스케일 결과 제출
- 22일차: 최종 PDF 제출
- 매일 23:59까지 제출
- 지각 3회 이상 시 완주 보상 제외

### CTA

- 참가 전: 참가하기
- 참가 후: 오늘의 미션 보기
- 결제 대기 상태: 결제 확인 대기
- 종료 상태: 결과 보기

## 5.6 오늘의 미션 화면

### 목적

사용자가 오늘 해야 할 일을 정확히 알고 바로 제출합니다.

### 화면 구성

- 챌린지명
- 현재 일차
- 오늘의 미션명
- 제출 마감 시간
- 남은 시간
- 필수 제출 항목
- 내 제출 상태
- 제출하기 버튼

### 상태별 UI

| 상태 | 표시 방식 |
| --- | --- |
| 미제출 | 제출하기 버튼 강조 |
| 제출 완료 | 검수 대기 배지 |
| 지각 제출 | 지각 제출 배지 |
| 승인 | 승인 배지 |
| 반려 | 반려 배지와 재제출 버튼 |
| 추가 확인 | 추가 확인 배지 |

## 5.7 제출 작성 화면

### 목적

사용자가 미션에 맞는 자료를 제출합니다.

### 공통 입력

- 제출 제목
- 설명 또는 본문
- 링크 입력
- 캡처 이미지 업로드
- PDF 파일 업로드
- 제출 버튼

### 챌린지별 표시

| 챌린지 | 필수 입력 |
| --- | --- |
| AI 동화 1-20일차 | 제목, 본문 또는 이미지/캡처 |
| AI 동화 21일차 | 업스케일 캡처 또는 결과 파일 |
| AI 동화 22일차 | PDF 파일 |
| 블로그 | 블로그 URL, 작성 완료 캡처 |
| 유튜브 | 유튜브 URL, 업로드 완료 캡처 |

### 업로드 UI

- 드래그 앤 드롭 영역
- 파일 선택 버튼
- 업로드된 파일 썸네일
- 파일 삭제 버튼
- 용량 초과 오류
- 형식 오류

### 제출 전 검증

- 필수 링크 누락
- 필수 캡처 누락
- PDF 누락
- 잘못된 URL 형식
- 파일 용량 초과

## 5.8 제출 상세 화면

### 목적

제출한 결과물과 검수 상태를 확인합니다.

### 주요 요소

- 제출 제목
- 제출 일시
- 정시/지각 여부
- 제출 링크
- 첨부 이미지
- 첨부 PDF
- 검수 상태
- 관리자 점수
- 반려 사유
- 재제출 버튼

## 5.9 랭킹 화면

### 목적

사용자의 경쟁/완주 동기를 높입니다.

### 주요 요소

- 내 순위 요약
- 전체 순위 테이블
- 닉네임
- 총점
- 승인 제출 수
- 정시 제출 수
- 좋아요 수
- 관리자 점수 합계

### 랭킹 UI

상위 3명은 강조 카드로 표시하고, 나머지는 테이블로 표시합니다.

## 5.10 마이페이지

### 목적

사용자가 자신의 챌린지 상태, 제출, 보상, 계좌 상태를 한곳에서 확인합니다.

### 화면 구성

1. 내 프로필 요약
2. 참여 중인 챌린지
3. 제출 내역
4. 보상 상태
5. 계좌 상태

### 보상 상태 표시

- 지급 예정
- 지급 보류
- 지급 완료
- 지급 제외

## 5.11 계좌 등록 화면

### 목적

현금성 보상을 받을 계좌 정보를 등록합니다.

### 입력 항목

- 예금주명
- 은행명
- 계좌번호
- 연락처
- 개인정보 수집 및 이용 동의

### 상태 표시

- 미등록
- 검수 대기
- 승인
- 반려

### UX 기준

- 계좌번호는 입력 후 마스킹합니다.
- 저장 후에는 뒤 4자리만 보여줍니다.
- 반려 시 반려 사유를 표시합니다.

## 6. 관리자 화면 설계

## 6.1 관리자 대시보드

### 목적

운영자가 오늘 처리해야 할 일을 한눈에 확인합니다.

### 핵심 지표

- 진행 중인 챌린지 수
- 전체 참가자 수
- 유료 참가자 수
- 검수 대기 제출 수
- 추가 확인 제출 수
- 지급 보류 보상 수
- 계좌 검수 대기 수

### 대시보드 카드

- 오늘 검수 대기
- 오늘 지각 제출
- 보상 지급 대기
- 계좌 승인 대기

## 6.2 챌린지 관리 화면

### 목록 정보

- 챌린지명
- 유형
- 상태
- 기간
- 참가비
- 참가자 수
- 제출 수
- 생성일

### 챌린지 생성 입력

- 챌린지 제목
- 챌린지 유형
- 설명
- 시작일
- 종료일
- 일일 마감 시간
- 참가비
- 유료 여부
- 보상 설명
- 상태

### 미션 설정

AI 동화 챌린지는 기본 22일 템플릿을 제공합니다.

- 1-20일차: 2쪽 작성
- 21일차: 전체 페이지 업스케일
- 22일차: PDF 제작 및 출판 준비

블로그/유튜브 챌린지는 일수와 필수 링크/필수 캡처 여부를 관리자가 설정합니다.

## 6.3 참가자 관리 화면

### 목적

참가자와 결제 상태를 관리합니다.

### 테이블 컬럼

- 닉네임
- 이메일
- 챌린지명
- 참가일
- 결제 상태
- 제출 수
- 승인 수
- 지각 수
- 완주 여부

### 액션

- 결제 상태 변경: pending, paid
- 참가 상태 변경: active, completed, dropped
- 사용자 제출 내역 보기

## 6.4 제출 검수 화면

### 목적

운영자가 제출물을 빠르게 검수합니다.

### 필터

- 챌린지 유형
- 챌린지명
- 제출 상태
- 정시/지각
- 미션 일차
- 제출일

### 제출 목록 컬럼

- 제출자
- 챌린지명
- 미션 일차
- 제출 시간
- 정시/지각
- 상태
- 첨부 수
- 링크 여부

### 제출 상세 패널

- 제출 제목
- 본문
- 링크
- 캡처 이미지
- PDF 파일
- 제출 시간
- 반려 이력

### 검수 액션

- 승인
- 반려
- 추가 확인
- 관리자 점수 입력
- 반려 사유 입력

## 6.5 랭킹 관리 화면

### 목적

### 테이블 컬럼

- 순위
- 닉네임
- 총점
- 승인 제출 수
- 정시 제출 수
- 좋아요 수
- 관리자 점수 합계
- 완주 여부

### 액션

- 제출 상세 보기
- 관리자 점수 수정
- 랭킹 재계산

## 6.6 보상 관리 화면

### 목적

페이백, 포인트, 상금 지급 상태를 관리합니다.

### 필터

- 관리자 화면은 모바일 최적화 우선순위를 낮게 둡니다.

## 8.2 태블릿

- 챌린지 카드는 2열
- 관리자 테이블은 가로 스크롤 허용

## 8.3 데스크톱

- 챌린지 카드는 3열
- 관리자 화면은 좌측 사이드바 + 우측 콘텐츠 구조
- 테이블과 상세 패널을 함께 볼 수 있는 2분할 레이아웃 권장

## 9. UX 정책

## 9.1 제출 UX

- 사용자는 오늘 제출해야 할 항목을 먼저 봅니다.
- 필수 항목이 빠지면 제출 버튼을 비활성화합니다.
- 제출 완료 후에는 상태 화면으로 이동합니다.
- 반려된 제출은 재제출 버튼을 명확히 표시합니다.

## 9.2 마감 UX

- 마감 시간이 3시간 이하로 남으면 마감 임박 배지를 표시합니다.
- 마감 이후 제출은 지각 배지를 표시합니다.
- 지각 제출은 승인 가능하지만 정시 점수를 받지 못한다는 안내를 표시합니다.

## 9.3 보상 UX

- 보상은 과장하지 않고 상태 중심으로 표시합니다.
- 지급 예정, 지급 보류, 지급 완료, 지급 제외를 명확히 구분합니다.
- 현금성 보상은 계좌 승인 필요 안내를 함께 표시합니다.

## 9.4 관리자 검수 UX

- 검수 대기 제출물을 기본 정렬로 보여줍니다.
- 승인/반려/추가 확인 버튼은 제출 상세 화면에 고정합니다.
- 반려 선택 시 반려 사유 입력 없이는 저장할 수 없습니다.
- 추가 확인 상태는 랭킹과 보상에서 제외된다는 안내를 표시합니다.

## 13. 최종 정리

프론트엔드의 핵심은 사용자가 **오늘 무엇을 해야 하는지**와 관리자가 **무엇을 검수해야 하는지**를 바로 알게 만드는 것입니다.

첫 디자인은 화려한 랜딩페이지보다, 챌린지 참여와 제출, 관리자 검수, 랭킹/보상 확인이 잘 되는 실사용 화면을 우선합니다.
